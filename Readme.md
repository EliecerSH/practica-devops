# Proyecto DevOps

## Descripción
Este repositorio demuestra el uso de GitFlow mediante ramas de características.

## Estrategia de ramas
Se utilizan Feature Branches:
- feature/frontend
- feature/docker
- feature/docs

Cada funcionalidad se desarrolla de manera aislada y luego se integra mediante Pull Requests hacia la rama main.

## Propósito
Simular un flujo de trabajo profesional con:
- HTML básico
- Docker con nginx
- Documentación del proyecto
